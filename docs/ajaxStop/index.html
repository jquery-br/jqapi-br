.ajaxStop()

Categories: Ajax > Global Ajax Event Handlers

.ajaxStop( handler() ) Retorna: jQuery

Descição: Registra um handler para ser chamado quando todas as requisições Ajax terminarem. Isso é um evento do Ajax.

  • adicionado na versão: 1.0.ajaxStop( handler() )

    handler()A função a ser invocada.

Sempre que uma requisição Ajax termina, o jQuery checa se existem ainda outras requisições pendentes. Se não restar nenhuma, o jQuery dispara o evento ajaxStop. Todos e quaisquer handlers que foram registrados com o método .ajaxStop() são executados nesse momento. O evento ajaxStop também é disparado se a última requisição Ajax pendente for cancelada devido ao callback beforeSend retornar false.

Para observar este método em ação, podemos configurar uma requisição Ajax básica:

<div class="trigger">Trigger</div>
<div class="result"></div>
<div class="log"></div>

Nós podemos vincular nosso handler do evento a qualquer elemento:

$('.log').ajaxStop(function() {
  $(this).text('Triggered ajaxStop handler.');
});

Agora podemos fazer uma requisição Ajax usando qualquer método jQuery:

$('.trigger').click(function() {
  $('.result').load('ajax/test.html');
});

Quando o usuário clicar no botão e a requisição Ajax terminar, a mensagem de log será mostrada.

Nota: Como .ajaxStop() é implementado como um método de instância jQuery, nós podemos usar a palavra-chave this para referenciar aos elementos selecionados a partir da função de callback.

Exemplo:

Esconde a mensagem de carregamento depois que todas as requisições Ajax terminarem.

$("#loading").ajaxStop(function(){
      $(this).hide();
      });
