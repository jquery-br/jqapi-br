.clone()

Categories: Manipulation > Copying

.clone( [ withDataAndEvents ] ) Returns: jQuery

Description: Cria uma cópia profunda da coleção dos elementos encontrados.

  • version added: 1.0.clone( [ withDataAndEvents ] )

    withDataAndEventsUm Booleano indicando se os eventos associados deverão ser copiados juntamente aos elementos. Assim como no jQuery 1.4, o conteúdo do elemento será igualmente copiado.

  • version added: 1.5.clone( [ withDataAndEvents ], [ deepWithDataAndEvents ] )

    withDataAndEventsUm Booleano indicando se os eventos associados deverão ser copiados juntamente aos elementos. O valor padrão é false. *Para 1.5.0 o valor padrão é incorretamente true. Isso será desfeito para false a partir da versão 1.5.1 em diante.

    deepWithDataAndEventsUm Booleano indicando se os eventos associados e os dados de todos os filhos, dos elementos clonados, devem ser copiados. Por padrão ele assume o valor informado ao primeiro parâmetro (que é por padrão false).

O método .clone() executa uma cópia profunda da coleção de elementos encontrados, gerando uma cópia sua e de todos os seus elementos filhos e nós de texto. Quando usado em conjunto com algum método de inserção, .clone() é uma froma conveniente de duplicar elementos numa página. Considere o HTML a seguir:

            <div class="container">
              <div class="hello">Hello</div>
              <div class="goodbye">Goodbye</div>
            </div>

Como mostrado o .append(), normalmente quando um elemento é inserido em alguma parte do DOM, ele é movido de sua posição anterior. Então, dado o código:

$('.hello').appendTo('.goodbye');

O resultado da estrutura do DOM seria:

            <div class="container">
              <div class="goodbye">
                Goodbye
                <div class="hello">Hello</div>
              </div>
            </div>

Para previnir isso e, ao invés, criar uma cópia do elemento, você pode escrever o seguinte:

$('.hello').clone().appendTo('.goodbye');

Isso irá gerar:

            <div class="container">
              <div class="hello">Hello</div>
              <div class="goodbye">
                Goodbye
                <div class="hello">Hello</div>
              </div>
            </div>

Note que ao usar o método.clone(), você pode modificar os elementos clonados ou seu conteúdo antes de (re-)inserí-los no documento.

Normalmente, quaisquer eventos associados ao elemento original não serão copiados para o clone. O parâmetro opcional withDataAndEvents nos permite alterar esse comportamento, e fazer cópias, também, de todos os eventos dos elementos associados à nova cópia do elemento. Como no jQuery 1.4, todos os dadso do elementos (atribuídos pelo método .data()) é também copiado para a nova cópia.

No entanto, objetos e arrays dentro dos dados do elemento não serão copiados e continuarão à ser compartilhados entre o objeto clonado e o objeto original. Para efetuar cópias profundas de todos os dados, copie cada um manualmente:

var $elem = $('#elem').data( "arr": [ 1 ] ), // Elemento original com dados apensos
              $clone = $elem.clone( true )
              .data( "arr", $.extend( [], $elem.data("arr") ) ); // Cópia profunda para evitar compartilhamento de dados

Como no jQuery 1.5, withDataAndEvents pode ser opcionalmente melhorado com o deepWithDataAndEvents para copiar os eventos e dados para todosos filhos do elemento clonado.

Examplo:

Clona todos os elementos b (e seleciona os clonados) e inseri-os após todos os parágrafos.

              
                <!DOCTYPE html>
                  <html>
                  <head>
                    <script src="http://code.jquery.com/jquery-git.js"></script>
                  </head>
                  <body>

                    <b>Hello</b><p>, how are you?</p>

                    <script>
                      $("b").clone().prependTo("p");
                    </script>

                  </body>
                </html>
