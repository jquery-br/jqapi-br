deferred.pipe()

Categories: Deferred Object

deferred.pipe( [ doneFilter ], [ failFilter ] ) Retorna: Promise

Descrição: Método utilitário para filtrar e/ou encadear objetos Deferreds.

  • adicionado na versão: 1.6deferred.pipe( [ doneFilter ], [ failFilter ] )

    doneFilter Uma função opcional que é chamada quando o objeto Deferred é resolvido.

    failFilter Uma função opcional que é chamada quando o objeto Deferred é rejeitado.

O método deferred.pipe() retorna um novo objeto promise que filtra os status e valores de um objeto deferred por meio de uma função. As funções doneFilter e failFilter filtram os status e valores originais do objeto deferido dos objetos resolvidos / rejeitados. Essas funções de filtro podem retornar um novo valor a ser passo por meio dos callbacks done() ou fail() ligados ao objeto promise ou eles podem retornar um outro objeto observável (Deferred, Promise, etc) que passará o seu status resolvido / rejeitado e valores para os calbacks dos objetos promise ligados. Se a função de filtro usada é null ou não especificada, o objeto promise ligado serv resolvido ou rejeitado com os mesmos valores que o original.

Examples:

Exemplo: Filtra o valor resolvido: 


var defer = $.Deferred(),
    filtered = defer.pipe(function( value ) {
      return value * 2;
    });

defer.resolve( 5 );
filtered.done(function( value ) {
  alert( "O valor é ( 2*5 = ) 10: " + value );
});

Example: Filtra o valor rejeitado: 


var defer = $.Deferred(),
    filtered = defer.pipe( null, function( value ) {
      return value * 3;
    });

defer.reject( 6 );
filtered.fail(function( value ) {
  alert( "O valor é ( 3*6 = ) 18: " + value );
});

Exemplo: Chain tasks: 


var request = $.ajax( url, { dataType: "json" } ),
    chained = request.pipe(function( data ) {
      return $.ajax( url2, { data: { user: data.userId } } );
    });

chained.done(function( data ) {
  // a informação recebida de url2 como provida da primeira requisição
});
