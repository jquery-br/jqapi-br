.focus()

Categorias: Eventos > Eventos de formulários | Formulários

.focus( handler(eventObject) ) Retorna: jQuery

Descrição: Atrela um manipulador de evento ao evento JavaScript "focus", ou dispara o evento em um elemento.

  • criado na versão: 1.0 .focus( handler(eventObject) )

    handler(eventObject) Função a ser executada toda vez que o evento ocorre.

  • criado na versão: 1.4.3 .focus( [ eventData ], handler(eventObject) )

    eventData Mapa de dados a serem passados ao manipulador do evento.

    handler(eventObject) Função a ser executada toda vez que o evento ocorre.

  • criado na versão: 1.0 .focus()

  • Nos dois primeiros casos mostrados anteriormente este método é uma forma abreviada para o método .bind('focus', handler) e na terceiro para o método .trigger('focus').
  • O evento focus ocorre em um elemento quando ele perde o foco. Este evento aplica-se a um conjunto limitado de elementos, tais como elementos de formulário (<input>, <select>, etc.) e links (<a href>). Em navegadores recentes o evento pode ocorrer em todos os elementos desde que se declare explicitamente o atributo tabindex para ele. Um elemento pode ganhar o foco via comandos de teclado, tal como tecla Tab ou por clique do mouse em um ponto da página fora do elemento.
  • Elementos que ganham o foco são destacados pelo navegador, por exemplo: com uma linha pontilhada em torno do elemento. Usa-se o foco para determinar qual elemento receberá eventos relacionados ao teclado.

Considere a seguinte marcação HTML:

<form>
  <input id="target" type="text" value="Field 1" />
  <input type="text" value="Field 2" />
</form>
<div id="other">
  Dispara o evento
</div>

O evento pode ser atrelado ao primeiro campo input:

$('#target').focus(function() {
  alert('Chamado o manipulador do evento .focus()');
});

Clicando no primeiro campo do formulário ou entrando nele via tecla Tab o alerta é mostrado:

Chamado o manipulador do evento .focus()

Podemos também disparar o evento com um clique em qualquer elemento:

$('#other').click(function() {
  $('#target').focus();
});

Segundo este código disparando o evento o alerta é mostrado.

O evento focus não se propaga pelo efeito bolha (bubble) no Internet Explorer. Assim, scripts que dependem de delegação de evento para focus não funcionam de forma consistente em todos os navegadores. Contudo, a partir da versão jQuery 1.4.2 foi introduzida a alternativa de mapear focus para focusin com seus métodos para delegação de eventos, .live() e .delegate().

Disparar o foco em elementos invisíveis causa um erro no Internet Explorer. Certifique-se de chamar o método .focus(), sem parâmetros, em elementos visíveis.

Exemplo: Ativando foco. 

<!DOCTYPE html>
<html>
<head>
  <style>span {display:none;}</style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  <p><input type="text" /> <span>focus fire</span></p>
  <p><input type="password" /> <span>focus fire</span></p>
<script>
    $("input").focus(function () {
         $(this).next("span").css('display','inline').fadeOut(1000);
    });
</script>
</body>
</html>

Exemplo: O código a seguir desabilita escrita em campos de texto: 

$("input[type=text]").focus(function(){
  $(this).blur();
});

Exemplo: O código a seguir destina-se a dar foco a um campo com id 'login' tão logo a página seja carregada: 

$(document).ready(function(){
  $("#login").focus();
});
