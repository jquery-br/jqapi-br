jQuery.extend()

Categories: Utilities

jQuery.extend( target, [ object1 ], [ objectN ] ) Returns: Object

Descrição: Une o conteúdo de dois ou mais objetos ao primeiro.

  • version: 1.0jQuery.extend( target, [ object1 ], [ objectN ] )

    target Objeto que recebrá as novas propriedades se objetos adicionais forem passados ou que vai estender o namespace jQuery se ele for o único argumento.

    object1Objeto que contém propriedades adicionais a serem unidas.

    objectNObjeto adicional contendo propriedades a serem unidas.

  • versão: 1.1.4jQuery.extend( [ deep ], target, object1, [ objectN ] )

    deepSe true, o merge será de forma recursiva (cópia em profundidade)

    targetObjeto que receberá as novas propriedades.

    object1AObjeto que contém propriedades adicionais a serem unidas.

    objectNObjeto adicional contendo propriedades a serem unidas.

Quando nós fornecemos dois ou mais objetos para o $.extend(), propriedades de todos os objetos são adicionados ao target.

Se apenas um argumento é fornecido ao $.extend(), isso quer dizer que o objeto target foi omitido. Nesse caso, o próprio objeto jQuery é assumido como o target. Ao fazer isso, nó podemos adicioanr novas funções ao namespace jQuery. Isso pode ser útil para autores de plugins que desejam adicionar novos métodos ao jQuery.

Tenha em mente que o objeto target (primeiro argumento) será modificado, e sempre retornado pelo $.extend(). Se, contudo, nós queremos preservar ambos os objetos originais, podemos faze-lo passando um objeto vazio como target:

var object = $.extend({}, object1, object2);

O merge realizado pelo $.extend() não é recursivo por padrão. Se uma propriedade do primeiro objeto for um objeto ou um array, ela será completamente reescrita por uma propriedade com a mesma chave no segundo objeto. Os valores não são unidos. Isso pode ser visto no exemplo abaixo examinando os valores de banana. Contudo, passando true para o primeiro argumento da funçáo, os objetos seráo unidos recurcivamente.

propriedades undefined não serão copiadas. Contudo, propriedades inherited do objeto prototype serão copiadas.

Exemplos:

Example: Unindo dois objetos, modificando o primeiro. 


var object1 = {
  apple: 0,
  banana: {weight: 52, price: 100},
  cherry: 97
};
var object2 = {
  banana: {price: 200},
  durian: 100
};

$.extend(object1, object2);

Resultado:

object1 === {apple: 0, banana: {price: 200}, cherry: 97, durian: 100}

Example: unindo dois objetos recursivamente, modificando o primeiro. 


var object1 = {
  apple: 0,
  banana: {weight: 52, price: 100},
  cherry: 97
};
var object2 = {
  banana: {price: 200},
  lime: 100
};

$.extend(true, object1, object2);

Resultado:

object1 === {apple: 0, banana: {weight: 52, price: 200}, cherry: 97, lime: 100}

Example: Unindo settings e options, modificando settings. 

var settings = { validate: false, limit: 5, name: "foo" };
var options = { validate: true, name: "bar" };
jQuery.extend(settings, options);

Resultado:

settings == { validate: true, limit: 5, name: "bar" }

Example: Unindo defaults e options, sem modificar o defaults. Isso é comum no padrão de desenvolvimento de plugins. 

var empty = {}
var defaults = { validate: false, limit: 5, name: "foo" };
var options = { validate: true, name: "bar" };
var settings = $.extend(empty, defaults, options);

Resultado:

settings == { validate: true, limit: 5, name: "bar" }
empty == { validate: true, limit: 5, name: "bar" }
