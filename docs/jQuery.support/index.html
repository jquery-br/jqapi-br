jQuery.support

Categoria: Properties > Properties of the Global jQuery Object | Utilities

jQuery.support Retorna: Object

Descrição: Uma coleção de propriedades que representa a presença de diferentes características do navegador ou erros (bugs).

  • adicionado na versão: 1.3jQuery.support

Ao invés de usar $.browser para detectar o agente de usuário atual e alterar a página com base em qual navegador está sendo executado, é uma boa prática para deteção de característica. Isto significa que, antes da execução do código que depende de uma função do navegador, testamos para garantir que a funcionalidade funciona adequadamente. Para tornar esse processo mais simples, jQuery executa muitos destes testes e torna os resultados disponíveis para nós como propriedades do objeto jQuery.support.

Os valores de todas as propriedades de suporte são determinados usando a detecção de recurso (e não usar qualquer tipo de browser sniffing).

A seguir, estão alguns recursos que explicam como a detecção de característica funciona:

  • http://peter.michaux.ca/articles/feature-detection-state-of-the-art-browser-scripting
  • http://www.jibbering.com/faq/faq_notes/not_browser_detect.html
  • http://yura.thinkweb2.com/cft/

Enquanto o jQuery inclui um número de propriedades, os desenvolvedores devem se sentir livres para adicionar suas próprias como as suas necessidades ditarem. Muitas da propriedades do jQuery.supportsão bem baixo-nível, por isso são mais úteis para plugin e desenvolvimento do núcleo jQuery, ao invés de desenvolvimento dia-a-dia em geral. Desde que o jQuery exija esses testes internamente, eles devem ser executados em cada carregamento de página; por essa razão, esta lista é mantida curta e limitada a recursos do jQuery em si.

Os testes incluídos no jQuery.support são estes a seguir:

  • ajax é igual a true se o navegador é capaz de criar um objeto XMLHttpRequest.
  • boxModel é igual a true se a página está renderizando de acordo com W3C CSS Box Model (atualmente retorna falso no IE 6 e 7 quando estes estão em Quirks Mode). Esta propriedade é null até que o document ready ocorra.
  • changeBubbles é igual a true se o evento change borbulha a árvore DOM, conforme exigido pelo W3C DOM event model. (É atuamente falso no IE, e o jQuery simula borbulha).
  • checkClone é igual a true se um navegador clona corretamente o estado checked de radio buttons ou checkboxes em fragmentos do documento.
  • checkOn é igual a true se o valor default de um checkbox é "on" quando nenhum valor é especificado.
  • cors é igual a true se um navegador pode criar um objeto XMLHttpRequest e se o objeto XMLHttpRequest tem uma propriedade withCredentials. Para habilitar requisições cross-domain no ambiente que não suporta cors ainda, mas que hablilite requisições (windows gadget, etc) cross-domain XHR, defina $.support.cors = true;. CORS WD
  • cssFloat é igual a true se o nome da propriedade contendo o valor CSS float igual a .cssFloat, como definido na CSS Spec. (É atualmente falso no IE, ao invés, use styleFloat).
  • hrefNormalized é igual a true se o método .getAttribute() recupera o atributo hrefdos elementos inalterados, em vez de normalizá-lo em uma URL totalmente qualificada. (É atualmente falso no IE, as URLs são normalizadas).
    DOM l3 spec
  • htmlSerialize é igual a true se o navegador é capaz de serializar/inserir elementos <link> usando a propriedade dos elementos .innerHTML. (é atualmente falso no IE).
    HTML5 WD
  • leadingWhitespace é igual a true se o navegador insere conteúdo com .innerHTML exatamente como especificamente-provido, se conduzindo espaços em branco caracteres são preservados. (É atualmente falso no IE 6-8).
    HTML5 WD
  • noCloneChecked é igual a true se os elementos DOM clonados sobrescrevem o estado de .checked. (É atualmente falso no IE). (Adicionado no jQuery 1.5.1)
  • noCloneEvent é igual a true se elmentos DOM clonados são criados sem manipuladores de evento (isto é, se os manipuladores de eventos no elemento fonte não estão clonados). (É atualmente falso no IE).
    DOM l2 spec
  • opacity é igual a true se um navegador pode propriamente interpretar a propriedade de estilo opacidade. (É atualmente falso no IE, este usa o filtro alpha).
    CSS3 spec
  • optDisabled é igual a true se o elemento option dentre os elementos option selecionados não estão automaticamente marcado como desabilitado (disable).
    HTML5 WD
  • optSelected é igual a true se um elemento <option> que está selecionado por padrão tem uma propriedade selected funcionando.
    HTML5 WD
  • scriptEval() é igual a true se scripts inline são automaticamente avaliados e executados quando inseridos no documento usando método de manipulação padrão DOM como .appendChild() e .createTextNode(). (É atualmente falso no IE, este usa .text para inserir scripts executáveis).
    Nota: Antes do jQuery 1.5.1, o método scriptEval() era a propriedade estática scriptEval. A mudança para um método permite o teste ser deferido até o primeiro uso para prevenir violações de segurança inline-script.
    HTML5 WD
  • style é igual a true se o estilo inline para um elemento pode ser acessado atravez do atributo DOM chamado style, como requerido pela especificação DOM Level 2. Neste caso, .getAttribute('style') pode recuperar este valor; no Internet Explorer, .cssText é usado para este propósito.
    DOM l2 Style spec
  • submitBubbles é igual a true se o evento submit borbulha a árvore DOM, como requisitado pelo W3C DOM event model. (É atualmente falso no IE, e o jQuery simula o borbulhar).
  • tbody é igual a true se um elemento vazio <table> pode existir sem um elemento <tbody>. De acordo com a especificação HTML, este sub-elemento é opcional, portanto a propriedade deveria ser true em um navegador totalmente compatível. Se falso, nós devemos contar para o navegador injetar tags <tbody> implicitamente. (É atualmente falso no IE, que atualmente insere tbody se esta não está presente na string associada à innerHTML).
    HTML5 spec

Exemplos:

Exemplo: Retorna o box model para o iframe. 

<!DOCTYPE html>
<html>
<head>
  <style>
  p { color:blue; margin:20px; }
  span { color:red; }
  </style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  <p>
  </p>
<script>

    $("p").html("This frame uses the W3C box model: <span>" +
                jQuery.support.boxModel + "</span>");

</script>

</body>
</html>
