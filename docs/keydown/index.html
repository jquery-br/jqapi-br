.keydown()

Categories: Events > Keyboard Events

.keydown( handler(eventObject) ) Retorna: jQuery

Descrição: Atribui um manipulador ao evento Javascript "keydown" ou dispara aquele evento em um elemento.

  • adicionado na versão: 1.0.keydown( handler(eventObject) )

    handler(eventObject)Uma função para executar a cada vez que o evento é disparado.

  • adicionado na versãao: 1.4.3.keydown( [ eventData ], handler(eventObject) )

    eventDataUm mapa de dados que será passdo para o manipulador de evento.

    handler(eventObject)Uma funçãao para executar a cada vez que o evento é disparado.

  • adicionado na versão: 1.0.keydown()

Este método é um atalho para .bind('keydown', handler) na primeira e segunda variações, e .trigger('keydown') no terceiro.

O evento keydown é enviado a um elemento quando o usuário primeiro pressiona uma tecla. Ele pode ser incorporado a qualquer elemento, mas o evento é enviado apenas para o elemento que ele está em foco. Elementos focáveis podem variar entre os browsers, mais elementos de formulário pode sempre ser focados então são candidatos razoáveis para este tipo de evento.

Por exemplo, considere o HTML:

<form>
  <input id="target" type="text" value="Hello there" />
</form>
<div id="other">
  Trigger the handler
</div>

O manipulador de evento pode ser acoplado ao campo input:

$('#target').keydown(function() {
  alert('Handler for .keydown() called.');
});

Agora quando o ponto de inserção está dentro do campo, pressionando uma tecla sempre mostra o alerta:

Manipulador para .keydown() chamado.

Para disparar o evento manualmente, aplique .keydown() sem um argumento:

$('#other').click(function() {
  $('#target').keydown();
});

Depois que este código executa, clica no Disparar o manipulador também alertará esta mensagem.

Se pressionar a tecla em quanquer lugar precisar ser capturado (por exemplo, para implementar atalho global das teclas em uma página), é útil acoplar este comportamento ao objeto document. Por causa da bolha de eventos, toda tecla pressionada percorrerá seu caminho pelo DOM para o objeto document a menos que seja explícitamente parado.

Para determinar qual tecla foi pressionada, examine o objeto de evento que é passado para o manipulador de função. Enquanto os browsers usam diferentes propriedades para guardar esta informação, jQuery normaliza a propriedade .which então você pode seguramente usá-la para obter o código da tecla. Este código corresponde à tecla no teclado, incluindo códigos para teclas especiais como setas. Para capturar a entrada de texto real, .keypress() pode ser uma melhor escolha.

<html>
<head>

fieldset { margin-bottom: 1em; }

}

  white-space: pre;
  padding: 5px;
  font-family: monaco, monospace;
  font-size: .7em;
}

</style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  <form>
  <fieldset>
    <label for="target">Type Something:</label>
    <input id="target" type="text" />
  </fieldset>
</form>
<button id="other">
  Dispara o manipulador
</button>
<script type="text/javascript" src="/scripts/events.js"></script>
<script>
var xTriggered = 0;
$('#target').keydown(function(event) {
  if (event.keyCode == '13') {
     event.preventDefault();
   }
   xTriggered++;
   var msg = 'Manipulador para .keydown() chamado ' + xTriggered + ' vez(es).';
  $.print(msg, 'html');
  $.print(event);
});

$('#other').click(function() {
  $('#target').keydown();
});</script>

</body>
