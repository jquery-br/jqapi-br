.mousedown()

Categories: Events > Mouse Events

.mousedown( handler(eventObject) ) Retorna: jQuery

Descrição: Acopla um manipulador de evento para o evento JavaScript "mousedown", ou dispara aquele evento em um elemento.

  • adicionado na versão: 1.0.mousedown( handler(eventObject) )

    handler(eventObject)Uma função para executar a cada vez que o evento é disparado.

  • adicionado na versão: 1.4.3.mousedown( [ eventData ], handler(eventObject) )

    eventDataUm mapa de dados que serão passados para o manipulador de evento.

    handler(eventObject)Uma função para executar a cada vez que o evento é disparado.

  • adicionado na versão: 1.0.mousedown()

Este método é um atalho para .bind('mousedown', handler) na primeira variação, e .trigger('mousedown') na segunda.

O evento mousedown é enviado a um elemento quando o ponteiro do mouse está sobre o elemento e o botão do mouse é pressionado. Qualquer elemento HTML pode receber este elemento.

Por exemplo, considere o HTML:

<div id="target">
  Clique aqui
</div>
<div id="other">
  Dispara o manipulador
</div>

O manipulador de evento pode ser ligado a qualquer <div>:

$('#target').mousedown(function() {
  alert('Manipulador para .mousedown() chamado.');
});

Agora se nós clicarmos sobre este elemento o alert é mostrado:

Manipulador para .mousedown() chamado.

Nós podemos também disparar o evento quando um elemento diferente é clicado:

$('#other').click(function() {
  $('#target').mousedown();
});

Depois que este código executa, cliques sobre Disparar o manipulador também alertarão a mensagem.

O evento mousedown é enviado quando um botão do mouse é clicado. Para atuar apenas em botões específicos, podemos usar a propriedade de evento de objeto which . Nem todos os browsers suportam esta propriedade (ao invés disso, o Internet Explorer usa button), mas o jQuery normaliza a propriedade assim que é seguro usar em qualquer browser. O valor de which será 1 para o botão esquerdo, 2 para o botão do meio, ou 3 para o botão direito.

Este evento é muito útil para assegurar que o botão primário foi usado para ser uma operaçao de arrastar; se ignorado, resultados estranhos podem ocorrer quando o usuário experimenta usar um menu de contexto. Enquanto os botões do meio e direito podem ser detectados com essas propriedades, este não é confiável. No Opera e Safari, por exemplo, os cliques do botão direito do mouse não são detectáveis por padrão.

Se o usuário clica em um elemento, arrasta para fora e solta o botão, isto é ainda contado como um evento mousedown. Esta sequência de ações é tratado como um pressionar do botão "cancelar" na maioria das interfaces de usuário, então é normalmente melhor ysar o evento click a menos que saibamos que o evento mousedown seja preferível para uma situação particular.

Exemplo:

Mostra textos quando disparados os eventos mouseup e mousedown.

<!DOCTYPE html>
<html>
<head>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  <p>Pressione e solte o botão do mouse aqui.</p>

<script>
    $("p").mouseup(function(){
      $(this).append('<span style="color:#F00;">Mouse up.</span>');
    }).mousedown(function(){
      $(this).append('<span style="color:#00F;">Mouse down.</span>');
    });

</script>

</body>
</html>
