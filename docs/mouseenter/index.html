.mouseenter()

Categories: Events > Mouse Events

.mouseenter( handler(eventObject) ) Returna: jQuery

Descrição: Acopla um manipulador de evento para ser disparado quando o mouse se posiciona sobre ou dispara aquele manipulador a um elemento.

  • adicionado na versão: 1.0.mouseenter( handler(eventObject) )

    handler(eventObject) Uma função para executar a cada vez que o evento é disparado.

  • adicionado na versão: 1.4.3.mouseenter( [ eventData ], handler(eventObject) )

    eventDataUm mapa de informação que será passado para o manipulador de evento.

    handler(eventObject) Uma função para executar a cada vez que o evento é disparado.

  • adicionado na versão: 1.0.mouseenter()

Este método é um atalho para .bind('mouseenter', handler) nas duas primeiras variações e .trigger('mouseenter') na terceira.

O evento JavaScript mouseenter é proprietário do Internet Explorer. Por causa da utilidade geral do evento, jQuery simula este evento assim ele pode ser usado independentemente de browser. Este evento é enviado a um elemento quando o ponteiro do mouse passa sobre o elemento. Qualquer elemento HTML pode receber este evento.

Por exemplo, considere o HTML:

<div id="outer">
  Fora
  <div id="inner">
    Dentro
  </div>
</div>
<div id="other">
  Dispara o manipulador
</div>
<div id="log"></div>

O manipulador de evento pode ser ligado a qualquer elemento:

$('#outer').mouseenter(function() {
  $('#log').append('<div>Handler for .mouseenter() called.</div>');
});

Agora quando o ponteiro do mouse move sobre o <div> Fora, a mensagem é adicionada ao final de <div id="log">. Você também pode disparar o evento quando um outro elemento é clicado:

$('#other').click(function() {
  $('#outer').mouseenter();
});

Depois que este código executa, cliques sobre Dispara o manipulador também serão adicionados ao final da mensagem.

O evento mouseenter difere de mouseover no modo que ele lida com o bubbling de eventos. Se mouseover fosse usado neste exemplo, quando o ponteiro do mouse movesse sobre o elemento Dentro, o manipulador seria disparado. Este é normalmente um comportamento indesejável. O evento mouseenter, por outro lado, apenas dispara o evento quando o mouse se posiciona sobre o elemento ao qual ele é ligado, não um filho. Então neste exemplo, o manipulado é disparado quando o mous fica sobre o elemtno Fora, mas não sobre o elemento Dentro.

Exemplo:

Mostra textos quando disparados os eventos mouseenter e mouseout. o mouseover dispara quando o ponteiro se move sobre um elemento filho também, enquando o mouseenter dispara apenas quando o ponteiro se move sobre o elemento ligado.

<!DOCTYPE html>
<html>
<head>
  <style>
div.out {
width:40%;
height:120px;
margin:0 15px;
background-color:#D6EDFC;
float:left;
}
div.in {
width:60%;
height:60%;
background-color:#FFCC00;
margin:10px auto;
}
p {
line-height:1em;
margin:0;
padding:0;
}
</style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  
<div class="out overout"><p>mova seu mouse</p><div class="in overout"><p>mova seu mouse</p><p>0</p></div><p>0</p></div>

<div class="out enterleave"><p>mova seu mouse</p><div class="in enterleave"><p>mova seu mouse</p><p>0</p></div><p>0</p></div>


<script>
    var i = 0;
    $("div.overout").mouseover(function(){
      $("p:first",this).text("mouse sobre");
      $("p:last",this).text(++i);
    }).mouseout(function(){
      $("p:first",this).text("mouse fora");
    });

    var n = 0;
    $("div.enterleave").mouseenter(function(){
      $("p:first",this).text("mouse entra");
      $("p:last",this).text(++n);
    }).mouseleave(function(){
      $("p:first",this).text("mouse sai");
    });

</script>

</body>
</html>
