Seletor :nth-child()

Categories: Selectors > Child Filter

Seletor nth-child

adicionado na versão: 1.1.4jQuery(':nth-child(index/even/odd/equation)')

  • index
    O índice de cada filho, começando com 1, a string even ou odd, ou uma equação, ex: :nth-child(even), :nth-child(4n) )

Descricão: Seleciona todos os elementos que são os enésimos filhos.

Devido a implementação do :nth-child(n) ser estritamente derivada da especificação do CSS, o valor do n é baseado no 1 (indexado a partir do 1 e não do 0 como em arrays), significa que a conta começa no 1. Para todos os outros seletores o jQuery segue a indexação do JavaScript que é baseada no zero. Portanto, se um <ul> têm dois <li>, $('li:nth-child(1)') seleciona o primeiro <li> enquanto $('li:eq(1)') seleciona o segundo.

A pseudo classe :nth-child(n) é facilmente confundida com o :eq(n), apesar dos seus dois resultado serem drasticamente diferentes. Com o :nth-child(n), todos os filhos são contados, indiferente do que eles sejam, e o elemento especiífico só é selecionado se ele bater com o seletor anexado com o peseudo classe. Com o :eq(n) apenas o seletor anexado à pseudo classe é contado, não limitado aos filhos de outros elemento, e o (n+1)to (n é baseado no zero) é selecionado.

Mais discussão pode ser lida no W3C CSS specification.

Exemplo: Isso é um parque para ver como o seletor funciona com diferentes strings. Note que isso é diferente do :even e :odd que não consideram os pais e apenas filtram a lista de elementos. Entretanto o :nth-child, conta o indíce de cada filho relativo ao seu pai em particular. Em qualquer caso, é mais fácil ver do que explicar...
